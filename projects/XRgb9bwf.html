Summary
Mission
HMW help people set up and use their Lutron + Ring systems together easily to have a safer, smarter home?
Lutron sells smart lighting systems, which includes lighting controls, sensors, shades, and lighting fixtures.
Ring sells home security systems, which includes video doorbells, security cameras, sensors, and other devices.
Solution
A simple mobile experience that helps you discover, understand, setup, and troubleshoot your home automation at ease.
How it started ?
It was a long-requested feature to let your security system affect your lights.
Lutron dealers often also sells Ring, and we hear a lot of requests from them for the integration. In 2020, Lutron and Ring business teams agreed to provide it first with Ring's security cameras and camera doorbells. More specifically to signal Lutron when doorbell was ringed or motion was detected.
Our Process
With a tight schedule, we took a LEAN approach

We only have 8 weeks and time for 1 round of user research in order to meet the Amazon Prime Day release. So we took a LEAN approach, started to develop concepts even before the interviews based on assumptions . And we conducted the user research as a combination of generative learning and validation.

1. Build empathy by using Ring products
We used Ring doorbell and cameras ourselves to build empathy. A key insight gained from this activity is that, motion detection can get overly sensitive under certain situations like a windy weather.

2. Online research
We found online that people are already hacking Lutron and Ring together with IFTTT. The most popular function is to activate a Lutron scene with Ring doorbell.

3. Hypothetical persona and user journey
We gathered persona information from existing research. Then we developed a user journey based on assumptions, which were captured as things to validation in the user research, along with other questions and unknowns.

4. Develop setup concepts
Guided by the lean approach, and to get some validation from real users, we developed 2 concepts for the setup experience (task analysis, information architecture design, and wireframing). We chose setup not only because it's the most fundamental function, but also because Lutron App has more precedence to reference for different kinds of setup.

Then we conducted 6 online user interviews

Generative learnings
1. Users want to known what an integration does before adding it
This is a pain point we learned about Lutron's existing integrations. Users can't easily figure out what an integration can do until after they added it. Users want that information upfront because it helps them decide whether to add the integration or not.
2. Users are in an "exploration mindset" when they are setting it up
Users don't decide everything upfront or have a clear goal in mind while setting up. Instead, it's a continuous process. Users explore what the feature can do and decide what to set up as they do it. This is because smart lighting and security is a newer concept for most people. This means our design needs to explain the capability and inspire user in an integrated way.
3. Security is the fundamental value proposition and social aspect is a key factor
We learned about the value propositions through showing different use cases. Some of our brainstormed ideas are proved to be undesired because they harm sense of security, even though they reduce annoyance.
On the other hand, ideas that may annoys the neighbor/visitors are also not welcomed even if they enhance security.

Design Process
User Story Map
Brainstorm and converge on user story map in phases
After another round of ideation, we prioritized the ideas with stakeholders based on user value and task size, and developed a phased implementation plan.
Example Decision: Select lights instead of scenes
Even though we found activating a scene is more popular on IFTTT than lights, through user interviews, we actually learned the opposite. Users prefer to select lights because otherwise it's more work to create a scene first. Our assumption is, on IFTTT people are actually preferring doorbells over motion detection, and it's harder to select and maintain multiple lights from a 3rd-party.

03
Final Design
1. Discover
User discovers the integration through notification when both Ring App and Lutron App are installed. (Phase II)
User gets an idea of what the integration is and get interested through viewing example use cases, along with key value propositions. User can also get inspired from the examples.
2. Setup
3 steps to setup an automation
1. Select a device
2. Select a trigger
3. Select lights and save
After setup: a clear overview
On both the device page and the homepage, user can see a summary of the setting.
On homepage, user can show only devices configured with the current Lutron system to get a clearer overview. This also helps user filter out devices at other homes (e.g., vacation homes, parents' home under the same account etc)
3. Troubleshoot
Identify whether the integration was triggered
Resolve issue by adjusting sensitivity on Ring App or snooze for a short time.
